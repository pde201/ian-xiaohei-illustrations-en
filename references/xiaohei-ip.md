# Xiaohei IP

## Character Definition

Xiaohei is the recurring visual character. Xiaohei is not a mascot sticker or decoration. Xiaohei is a serious operator inside the concept.

Every generated image should include Xiaohei unless the user explicitly asks otherwise, or unless dense architecture mode is selected and Xiaohei would reduce clarity.

## Appearance

- Small solid-black creature.
- White dot eyes.
- Tiny thin legs, optionally tiny thin arms.
- Body can look like a bean, blob, cylinder, black box, funnel, shadow, or machine part.
- Slightly uneven hand-drawn outline.
- Blank, deadpan, serious expression.

## Personality

- Serious about the work.
- Doing something slightly absurd but conceptually correct.
- Quiet system operator energy.
- Deadpan, not cute.
- A little awkward, not foolish.

## Useful Jobs

Make Xiaohei perform the core conceptual action:

- carrying source material
- pulling a boundary line
- getting stuck in a system breakpoint
- operating a decision lever
- becoming a filter or funnel
- sorting evidence
- stamping an output
- holding open a gate
- turning a valve
- weighing inputs
- repairing a broken flow
- recording lineage
- guarding against a bad path

## Avoid

- Do not make Xiaohei overly cute.
- Do not make Xiaohei a children's cartoon character.
- Do not add complex clothing, expressive faces, sparkly eyes, or mascot poses.
- Do not leave Xiaohei standing in the corner.
- Do not let Xiaohei overpower the concept.
- Do not draw Xiaohei as polished commercial character art.

## Test

If removing Xiaohei leaves the metaphor fully intact, Xiaohei is too decorative. Rewrite the prompt so Xiaohei performs the action that explains the idea.
