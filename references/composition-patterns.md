# Composition Patterns

Choose one structure. Do not mix too many patterns in one image.

## Workflow

Use for input -> process -> output, content creation, automation, or data movement.

Draw inputs on one side, Xiaohei or a strange machine in the center, and outputs on the other side. Use orange for the main movement.

## System Detail

Use for data sources, filters, databases, renderers, agent systems, or product boundaries.

Draw only 3-5 core parts. Xiaohei should operate, guard, connect, or repair one crucial part.

## Before / After

Use for chaos/order, manual/automated, scattered/consolidated, anxious/stable, blocked/working.

Draw the messy state and the resolved state with a restrained orange transition. Keep each side simple.

## Character State

Use for user pain, team workflow pressure, too many tools, information anxiety, or getting unstuck.

Draw 2-4 small states. Each state should have one short label.

## Concept Metaphor

Use for product factories, information warehouses, black boxes, workflow machines, audit trails, or decision engines.

Draw one strange low-tech object with a few inputs and one output. Make it memorable without making it dense.

## Method Layers

Use for frameworks, stacks, capability layers, or reusable foundations.

Draw layered shelves, boxes, drawers, ladders, or scaffolding. Avoid a formal pyramid. Make Xiaohei build, support, or bolt the layers together.

## Map Route

Use for paths from idea to launch, user journeys, learning routes, and handoffs.

Draw a single winding route with a few labeled stops. Xiaohei should pull, carry, or walk the path.

## Comic Strip

Use for failure-to-success, real process, friction, or small narrative turns.

Draw 2-4 small scenes. Each scene gets one action and one short label.

## Original Metaphor Method

1. Convert the abstraction into a physical action: stuck, leaking, sorting, weighing, folding, unpacking, rerouting, stamping, filtering, compressing, balancing.
2. Convert the system into a low-tech object: broken machine, drawer, pipe, post office, scale, gate, shelf, valve, press, ledger, ladder, suitcase, bucket, scanner.
3. Make Xiaohei perform the action: pulling, carrying, stuffing, fishing out, pressing, weighing, sewing, cutting, turning, guarding, pushing, catching, stamping, repairing.

Use only 1-2 major objects. Do not fill the canvas with clever props.

## Anti-Repetition Rule

Do not copy old examples or repeat stock compositions. If a similar theme appears again, change the physical metaphor and Xiaohei action.

Examples:

- A handoff path does not always need a road; it could be a door latch, conveyor belt, or mail slot.
- A shared engine does not need a stack diagram; it could be a heavy block lifting several shelves.
- A data boundary does not need boxes and arrows; it could be a valve, lock, filter, or quarantine tray.
