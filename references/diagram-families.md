# Diagram Families

Pick one family per image.

## Workflow

Input -> transformation -> output with one primary path.

## Sequence

Time-ordered interactions across 2-6 actors. Keep message labels short.

## State

Finite states and transitions. Highlight entry/exit and failure loops.

## Deployment

Runtime topology across services, data stores, and environments/zones.

## Data Lineage

Source -> transform -> serve with quality/verification checkpoints.

## Incident Timeline

Chronological event chain with trigger, impact, mitigation, and recovery.

## Decision Tree

Branching logic with bounded depth. Label branch conditions precisely.

## Runbook Flow

Operator-first procedural flow with checks, gates, and escalation path.

## Family Selection Heuristics

- Architecture/system requests: deployment, sequence, data lineage.
- Reliability/ops requests: incident timeline, runbook flow, state.
- Strategy/process requests: workflow, decision tree.
